# ru-catfishing — баланс дней + надёжный зачёт (дизайн)

Дата: 2026-06-29. Автор сессии: Amal + Claude.

## Цель
По итогам 25 дней:
1. **Баг зачёта** — `norm()` рвёт ответы на пунктуации (апострофы/дефисы внутри
   слова), требует совпадения мультимножества токенов, не прощает хвостовые
   уточнители и ведущие частицы. Кейсы от Саши Потапенко: «Извержение Везувия» vs
   «Извержение Везувия в 79 году»; «Дивный новый мир» без «О». Мой кейс: «Жанна
   д'Арк» → «жанна дарк».
2. **Перекос тематики/сложности** — `bulk_build.py` шаффлит пул Featured/Good и
   режет по 10 подряд → кластеры (много игр подряд) и случайная сложность.

## Решения (выбраны с пользователем)
- Структура дня: **микс + мягкая квота** — max 2 пазла одной темы, min 5 разных
  тем, кривая сложности **3🟢 / 4🟡 / 3🔴**. Порядок внутри дня — **вперемешку**
  (не сортируем по сложности, не реверсим; детерминированный шаффл, без длинных
  серий одного тира).
- Метрика сложности: **pageviews (Wikimedia REST) + число категорий**, тиры по
  перцентилям пула.
- Источник пула 26+: **гибрид** — переиспользуем уже собранные дни 26–123 +
  курируемый список известных тем + дедуп против **всех** дней -1…25.
- Зачёт: **умеренная лояльность** (см. ниже). Гард-рейл «не генерить короткую
  форму, если осталось <2 слов» — НЕ нужен. Вместо него — кнопка отзачёта.

## Секция A — зачёт
Модель приватности не меняется: клиент хеширует ввод, сверяет с `accept:[sha256]`.

### A1. norm() (байт-идентично в make_day.py и +page.svelte; после правки —
пересборка ВСЕХ дней, хеши зависят от norm)
1. Удалять апострофы/кавычки `' ’ ‘ ʼ \`` **без пробела**, ДО шага «пунктуация→
   пробел» → «д'Арк»→«дарк».
2. Расширить стоп-слова с `{the, of}` до
   `{the, of, о, об, обо, и, а, но, в, во, на, не}` (симметрично с обеих сторон,
   только смягчает) → «О дивный новый мир» == «дивный новый мир».

### A2. expand_forms() (Python, build-side)
Генератор «короткой формы»: отрезает хвостовой уточнитель по regex
(`\s+в\s+\d{1,4}\s+году?$`, `\s+\d{3,4}\s+года$`, `,?\s+\d{3,4}$`). Применяется к
title И к каждому редиректу/langlink; каждый вариант → norm → sha256 в accept.
(Скобки `norm` и так срезает.) Редиректы/langlink/фамилия уже идут через norm —
автоматически получают A1.

### A3. Кнопка отзачёта «Я не это имел в виду»
Зеркало к существующей «Я всё-таки прав» (wrong→half). После автозачёта (win)
игрок может отозвать: **win → miss**. Честная система в обе стороны.

## Секция B — классификация
Author-only метаданные пула (в `prototype/data/_pool_classified.json`), КРОМЕ
difficulty, который теперь шипается в strict per-puzzle.

### B1. Тема (по доменам, не по «персона/не-персона»)
10 тем: Кино и сериалы · Игры · Музыка · Искусство · Литература · Наука и техника
· История и события · Спорт · География и природа · Общество/Прочее (fallback).
Классификатор — приоритетный список ключевых слов по строкам категорий статьи.

### B2. Сложность → тир 🟢/🟡/🔴
- pageviews (главный): Wikimedia REST per-article, среднее помесячное за 12 мес,
  кэш `_pageviews_cache.json`.
- число полезных категорий (вторичный).
- score ≈ log(pageviews) + бонус за категории; тиры по перцентилям пула.

### B3. Показ сложности на фронте
strict per-puzzle получает поле `difficulty: "easy"|"medium"|"hard"`. На экране
ответа (после reveal, не до) рендерим «Сложность: 🟢 Лёгкий / 🟡 Средний / 🔴
Сложный». Фронт толерантен к отсутствию поля (старые сборки).

## Секция C — композиция дней
1. Locked-сет: все тайтлы дней -1…25 (по нормализованному тайтлу) — не
   переиспользуем; контент 0–25 не меняем (только перехеш под новый norm).
2. Пул-кандидат 26+: тайтлы 26–123 + курируемые известные − locked − внутренние
   дубли.
3. Жадная композиция (фикс seed): бакетим по (тема×сложность); на день набираем
   10 при кривой 3/4/3, max 2 темы, min 5 тем, выбор темы по least-recently-used
   по горизонту. Внутри дня — детерминированный шаффл (без длинных серий тира).
   Невыполнимое ограничение → мягкая релаксация + лог.
4. Выход: dayN.json (strict, +difficulty) + dayN.txt; gen_index.py → index.js.
   DQ: ≥4 категории, 0 per-puzzle утечек, 0 дублей по всем дням.
