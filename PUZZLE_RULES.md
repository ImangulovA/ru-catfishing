# Правила подбора загадок (ru-catfishing)

Игра: по списку категорий ru.wikipedia угадать статью. Хорошая загадка
**«раскручивается»**: по категориям можно дойти до одного-единственного ответа,
и в этот момент случается «ага!». Плохая загадка не сходится: под её категории
подходит слишком много статей, либо категорий просто мало.

Этот файл — источник истины по подбору. Любые новые дни сверяем с ним.

---

## Главный критерий: «раскручивается ли?»

Перед тем как оставить статью в дне, задай вопрос:

> Если бы я видел ТОЛЬКО эти категории (без названия), смог бы я сузить до
> ровно этой статьи, и было бы это приятно?

- Сходится к ~одному ответу + ответ узнаваемый → берём.
- Подходит много статей / категорий мало / ответ безликий → НЕ берём, заменяем.

---

## Как мы ДЕЛАЕМ (DO)

- **6+ полезных категорий** на пазл (после фильтрации). Чем больше зацепок,
  тем интереснее раскручивать.
- **Различающие, сходящиеся категории.** Нужны 2-3 узкие категории, которые
  вместе указывают почти на одну статью: конкретный автор / режиссёр / серия /
  художественное направление / конкретное событие. Не только широкие ведёрки.
- **Узнаваемый ответ.** Статья достаточно известная, чтобы «ага!» сработало.
  Малоизвестный ответ при общих категориях = нечестно.
- **Цепочка раскрутки.** Игрок должен мочь связать зацепки:
  «картина + 1937 + Пикассо + Гражданская война в Испании → Герника».
- **Разнообразие типов за день** (человек, фильм, книга, игра, место, событие,
  картина, животное), НО каждый по отдельности проходит проверку на сходимость.
- **Люди — с ярким многогранным профилем.** Лучше человек с необычным
  пересечением (например, актриса И изобретательница), чем рядовой представитель
  огромной профессии.

## Как мы НЕ делаем (DON'T)

- **Мало категорий** (< 4-5 полезных) → нечем дедуцировать.
- **Только общие категории с огромным членством** («Актрисы России»,
  «Хищные птицы», «Картины по алфавиту») → под них подходят тысячи статей.
- **«Один из многих» без сужающих категорий**: рядовая актриса, проходная
  картина, обычное животное — подходит под тысячи.
- **Слишком много вариантов**: набор категорий честно описывает десятки статей.
- **Год рождения/смерти как единственная сужающая зацепка.** Год — приятный
  бонус, но сам по себе не раскручивает.

## Отклонённые примеры (плейтест day1, 2026-06-05)

- **Белоголовый орлан** — мало категорий, слишком общие (любая хищная птица
  Северной Америки подходит). Не раскручивается.
- **Гробницы Атуана** — малоизвестная книга, категории не сходятся.
- **Демида, Валла Сергеевна** — одна из слишком многих актрис, ничем не
  выделяется, не раскручивается.
- **«Девушка, освещённая солнцем»** — проходная картина, нет отличительных черт.

## Процесс проверки (без открытия json)

1. Правим только сырой `prototype/data/dayN.txt` (одна статья на строку, в
   `.gitignore`). Строгий json не трогаем руками.
2. Гоним `fbpython scripts/make_day_from_txt.py N`.
3. Скрипт печатает **превью**: по каждому пазлу заголовок + число категорий +
   сами категории. Строгий json НЕ открываем — судим по превью.
4. По каждому пазлу применяем «раскручивается ли?». Помечено `LOW (<4)` —
   почти наверняка под замену.
5. Что не сходится — заменяем в txt и повторяем.

---

## Почему json не трогаем руками

Строгий `app/src/lib/days/dayN.json` — это массивы `sha256`-хешей и
`base64`-строк. Для сканеров безопасности это выглядит как утёкшие секреты, и
работа с таким файлом упирается в гардрейлы. Поэтому: редактируем txt, json
генерим скриптом и не открываем.
